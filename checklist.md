Check Current Branch
Before coding, confirm you’re on the correct Git branch (e.g., feature or bugfix). Avoid committing directly to main.


Plan First
Before implementation, create a brief plan or outline for the feature or fix you are adding.


Reference Documentation
Always consult project documentation, architecture, or roadmap files to ensure you align with existing guidelines.


Set AI Context
When using AI tools, explicitly mention the relevant files or reference materials they should follow.


Update Logs
After a coding session, update a log or notes file with a brief summary of what you’ve done.


Track To-Dos
Maintain a todo list for items deferred or skipped, ensuring nothing is forgotten.


Test Before Commit
Always run tests or validate functionality before making any commit.


Consistent Commit Messages
Use clear commit messages following a pattern (e.g., feat: add overlay drag, fix: resolve audio lag).


Branch Properly
Ensure you follow the branching strategy (develop, feature/bugfix branches), and merge through proper pull requests or reviews.


Keep Scope
Avoid expanding beyond the current scope—stick to the plan and defer extra ideas to your todo list.
